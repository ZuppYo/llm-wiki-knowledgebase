ลองไมค์ สัปดาห์นี้จะพาทุกคนมาปูพื้นฐานการสร้าง AI Agent ด้วย "LangChain" Framework ที่เปรียบเสมือนตัวกลางในการเชื่อมต่อ LLM เข้ากับ application และเครื่องมือต่างๆ ของเรากันครับ ในหัวข้อนี้เราจะมาเริ่มต้นทำความรู้จักกับ concept และองค์ประกอบหลักของ LangChain กันว่ามีอะไรอยู้บ้าง และองค์ประกอบเหล่านี้สามารถนำมาใช้ร่วมกับ LLM เพื่อพัฒนา application ได้อย่างไรบ้าง มาดูกันในหัวข้อลองไมค์วันนี้กันนะครับ  
  
โดยในหัวข้อนี้เราจะมาพูดถึงประเด็นเหล่านี้กัน  
\- ทำความรู้จักกับ LangChain และ concept การพัฒนา AI Agent ในปัจจุบัน  
\- การ Setup Model (Gemini) และลองเขียน code สั่ง LLM ด้วย Python  
\- ทำความเข้าใจประเภทของ Message (System, Human, AI, Tool) และการส่งรูปภาพ (Multimodal) ให้ AI วิเคราะห์  
\- การสร้าง Custom Tools และการเรียกใช้เครื่องมือที่ Model มีอยู่แล้ว (เช่น Google Search)  
\- การจัดรูปแบบคำตอบของ AI (Structured Output) ให้ออกมาเป็น JSON หรือ Data class เพื่อนำไปประยุกต์ใช้ต่อกับระบบอื่น

### 2\. Message - ตัวอย่างส่งภาพผ่าน Content ใน Message

**27:29** · ภาพครับแล้วผมก็เอาภาพ 1 เข้ามาครับเป็น ภาพน้องหมาตัวนี้อยู่ใน example.png PNG ตัวนี้นะครับเพราะฉะนั้นสิ่งที่ผมจะทำ ครับผมจะทำ 2 อย่างครับ 1 ครับผมจะทำการ สร้างฟังก์ชันในการอ่านภาพขึ้นมาซึ่งก็ คือฟังก์ชันตัวนี้ผมสร้างเป็นฟังก์ชัน อ่านภาพชื่อว่า Ncode Image ครับโจทย์ ของ Ncode image ก็คือผมจะรัด image เข้าไปและหลังจากนั้นครับก็จะทำการ return เป็น BAS 64 ออกมาซึ่งไอ้เบส 64 นี่แหละ ครับก็คือเป็นตัวที่เป็นเหมือนกับ Bary ในการที่จะทำการส่งข้อมูลไปยังตัวของ

**28:28** · document ของตัวที่เป็น multimel ได้มัน ก็มีหลากหลายวิธีในการส่งนะครับอ่ะเพราะ ฉะนั้นครับสิ่งที่ผมเปลี่ยนตอน message ตรงนี้ครับผมก็จะเปลี่ยนเป็น low user คราวนี้เราไม่ได้มีการดาวโหลด system พร้อมอะไรนะเราเอาแบบง่ายๆเลยจริงๆคุณจะ ใส่ system พร้อมเพิ่มก็ได้นะ Low user content อ่ะจากแต่เดิมที่เราเปลี่ยนเป็น text ไม่ใช่เปลี่ยนเป็น text เราใส่เป็น text เราก็จะบอกเป็น text 1 ก้อนว่าภาพ นี้คืออะไรแล้วก็ส่งเป็นภาพคู่ไปก็คือ เป็นเบส 64 ตัวนี้เพื่อให้มันตอบออกมาว่า

**28:49** · นี่คือภาพของอะไรเบส 64 ก็คือตัวที่เรามี การใช้คำสั่ง n code ในการที่จะแปลงภาพ เข้ามานะครับอ่ะนี่มันมีตัวแดงตรงเบ 64 นิดนึงเดี๋ยวขอ Import เข้ามาก่อนก็คือ import ตรงๆเลย Import เบ 64 เข้ามาแบบ นี้นะครับ 64 โเคเสร็จแล้วทุกคนอันนี้ก็ ไม่ต้องใส่แล้วเพราะเนื่องจากเราไม่ได้มี การ Import เรียกใช้librี่ของตัวที่เป็น message ตัวนั้นแล้วนะครับคอนเซปก็จะ ประมาณนี้เพราะฉะนั้นสิ่งที่เราทำครับเรา แค่เพิ่มคำสั่งในการอ่านภาพมาตรงตำแหน่ง นี้ให้กลายเป็นเบ 64 ก้อนนึงมาแล้วก็ส่ง เบ 64 ตัวนี้ใน message วิธีเดิมที่เรา

**29:35** · สายตาที่น่ารักนะครับประมาณนี้ทุกคนอ่ะ เพราะฉะนั้นครับ message เนี่ยมันมีลูก เล่นอยู่ประมาณนี้ก็ขึ้นอยู่กับว่าคุณจะ ดีไซน์ความเป็น message ยังไงออกมาซึ่ง ทุกคนนลองไปหาอ่านเพิ่มเติมได้เดี๋ยวผม ค่อยกลับมาเล่าอีกทีนึงในเรื่องของการ message ในการสร้าง agent นะครับแต่ว่า นี่ concept ที่ 2 และ framework agent

**42:48** · ที่พบอ่ะก็คือคุณอนุทินอ่ะก็จะมีการคืน ตรงนี้ออกมาท้ายสุด AI Message ก็จะตอบ มาว่านายกรัฐมนตรีคนปัจจุบันของประเทศไทย คือคุณอนุทินชานวีรกุลครับเพราะฉะนั้นคำ ถามก็คือแล้วคุณจะเอาข้อมูลตัวไหนมาตอบคำ ตอบก็คือคุณก็ต้องเอาข้อมูลเนี่ยเห็นมั้ย สิ่งที่เราจะเห็นมันเป็นเหมือนเนของการทำ งานเนอะมันจะเป็นสเต็ป 1 สเต็ป 2 สเต็ป 3 สต็ป 4 คุณก็ต้องเอาสเต็ปสุดท้ายมาแล้วก็ ต้องดึงข้อมูล content ของสเต็ปสุดท้าย ออกมานั่นคือเหอผลที่ว่าทำไมผมถึงใส่ -1.

**43:12** · content มาก็คือเอา message ก้อนสุดท้าย แล้วก็ D content ก็คือตรงตำแหน่งนี้ออก มาทำการต่อไปนั่นเองนะครับแล้วก็จริงๆอ่ะ ถ้าทุกคนลองดูใน message เมันก็จะมี responsะ data ที่เราเคยอธิบายเรื่อง meta data ไปครับมันจะมีการเขียนเอาไว้นะใน แต่ละสต็ปหรือเปล่าว่ามันมีการใช้งานยัง ไง Stop ถูกต้องดีมยแล้วก็มีการใช้ Token ไปเท่าไหร่เนี่ยทุกคนก็จะเห็น Userage Meta Data อะไรพวกนี้ออกมาได้ด้วยนะ ครับอ่ะโอเคนะเพราะฉะนั้นอันนี้คือหนึ่ง ในวิธีที่ทุกคนสามารถมา Bug ผ่าน Message ได้เมื่อผมลองรันอีกทีนึงนะโดยใช้แบบแค่

**43:40** · -1 message ตัวนี้ผมก็จะได้เพียงแค่แค่ ตัวที่เป็น message เนาะคำตอบออกมาได้ว่า นายกรัฐมนตรีคนปัจจุบันของเราก็คือคุณ อนุทินนั่นเองนะครับอ่ะทีนี้เมื่อกี้ผม บอกว่าทำไมผมถึงต้องกำหนด system พร้อม ได้ดูดันไม่เกรงใจใครขนาดนี้เนอะผมลองไม่ ส่ง system พร้อมไปครับผมลองส่งไปแค่นี้ นะ create agent กับ true ขึ้นมาครับ แล้วผมลองรันดูครับสิ่งที่เกิดขึ้นก็คือ เห็นมั้ยเจentเป็นกล้าว่างทุกคนคุณก็เฮ้ย คุณทำอะไรผิดป่ะเนี่ยเมื่อกี้เราบอกว่าไง ครับเราบอกว่า bug เนาะเราต้องทำการ debug แล้วเมื่อกี้ผมไม่น่าลบไปเลย respons เศษ อ่ะเราลองมาดูสิครับว่ามันเกิดอะไรขึ้น อ่ะถ้าเราลองดูนะผมทำการช็อตคให้ทุกคน

**44:12** · เห็นในสเต็ปนี้เลยเนี่ยทุกคนจะสังเกตเห็น เปล่าว่า AI message อันสุดท้ายอ่ะ content มันไม่ได้อะไรออกมาแต่ว่าถ้าคุณ ลองไปดูที่เครื่องมือครับคือเครื่องมือ อ่ะมันหาข้อมูลเจอนะแต่มันน่ะไม่ยอมเอา เครื่องมืออ่ะครับผลลัพธ์ผลลัพธจาก เครื่องมืออ่ะมาส่งต่อให้กับ AI agent ครับผมก็เลยค้นพบว่าอ้าวนี่ผมต้องกำหนด ผ่าน System ด้วยเหรอเนี่ยมันถึงจะยอมเอา มาใช้นะครับซึ่งเหตุการณ์เนี้ยเกิดขึ้น ได้นะครับและเวลาที่เราใช้ของจริงอ่ะถ้า

**44:35** · คุณเคยฮูดูหัวข้อ ADK ของผมอ่ะผมจะกำหนด System พร้อมไว้ดุมากนะครับจริงๆตอนใน หัวข้อนั้นเราก็เหมือนทำตาม document เ จริงจริงๆ document เค้าอ่ะถ้าคุณลองไปดู ตัวของหัวข้อ IDK ทุกคนจะค่อยค้นพบว่ามัน จะมีการกำหนดเลยว่าใช้เครื่องมืออะไรจาก เงื่อนไขอะไรผลลัพธ์ไม่ออกมาเป็นยังไงผล ลัพธส่งต่อเป็นยังไงมันจะมีการเขียนอะไร พวกนี้ไว้ชัดเจนมากนะครับเพราะฉะนั้น เนี่ยเป็นเหตุผลที่ว่าการใส่ System พร้อมนะอ่ะครับก็จะส่งผลทำให้คำตอบสุด ท้ายอ่ะมันสามารถที่จะออกมาอย่างถูกต้อง ได้นะครับเสร็จแล้วมันก็จะสามารถที่ควบ คุมสิ่งที่เรียกว่าโทนของการต่อมาได้ด้วย

**45:02** · คุณจะกำหนดผ่าน system พร้อมตัวนี้ก็ได้ หรือคุณจะไปกำหนดผ่าน System message จากการใส่ message Array ตัวนี้ก็ได้ ครับแล้วแต่วิธีการที่เราใช้เลยนะครับอ่ะ นี่นี่คือวิธีการสร้างเครื่องมือซึ่งใน กรณีที่คุณจะทำอ่ะพวกที่เป็น CAS Rag ต่างๆนะครับคุณสามารถที่จะใช้วิธีการนี้ ได้เลยนั่นก็คือคุณเปลี่ยนจากเนี่ยการ เสิร์ช database ที่เป็น database ปลอมๆ ให้เป็น database จริงๆได้คุณเปลี่ยนเป็น การเสิร์ช API จริงๆออกมาได้นะครับเพราะ ฉะนั้นอันเนี้ยขึ้นอยู่กับวิธีการดีไซน์ และแต่มันก็จะมีอีกเวย์นึงก็คือในกรณีที่

**45:29** · เฮ้ยแล้วถ้าเราต้องการดึงข้อมูลก่อนเสมอ ล่ะดึงจากเวกเตอร์ database ก่อนแล้วถึง จะค่อยมาส่งตัวนี้ซึ่งถ้าคุณอยากได้ control แบบเป๊ะๆยิ่งขึ้นมันก็จะมี 2 วิธี 1 ก็คือก็เหมือนกับเวลาที่เราเขียน โค้ดปกติถ้าใครที่ดูหัวข้อเวกเตอร์ database มาเนอะทำสิ่งนั้นก่อนค่อยเอา context มาคุยในนี้ก็ได้ก็คือเหมือน เขียนโค้ด Python ปกติกับอันที่ 2 ก็คือ คุณอาจจะใช้วิธีการของแรงกราฟในการควบคุม flow ในการที่จะส่งข้อมูลก็ได้นะครับอ่ะ

**46:47** · ลัพธ์เสิร์ชออกมาจริงๆมันได้เป็นjสัน formมatบางอย่างออกมาแหละเพียงแต่ว่า เพื่อให้เกิดความสวยงามในการดึงข้อมูลออก มาครับโดยปกติเรามักจะกำหนดผ่านตัวที่ ชื่อว่า response schema นั่นก็คือผล ลัพธ์ที่เราอยากได้เราอยากได้ schema หน้าตาประมาณไหนออกมาแล้วหลังจากนั้นเรา ก็จะกำหนด schema ในการรับข้อมูลซึ่งเป็น แบบเดียวกันกับ format ของ pantic นะครับ
