ลองไมค์ สัปดาห์นี้จะพาทุกคนมาทำความรู้จักกับ "LangGraph" เครื่องมือสำคัญอีกตัวสำหรับการพัฒนา AI Agent ที่มักนำมาใช้ทำงานคู่กับ LangChain ครับ ในหัวข้อนี้เราจะมาดูกันว่า LangGraph คืออะไร ทำไมถึงเกิดมาเพื่อตอบโจทย์การจัดการ Workflow ที่มีความซับซ้อน และมีจุดเด่นที่แตกต่างจาก LangChain ปกติอย่างไรบ้าง พร้อมพาไปทำความเข้าใจองค์ประกอบหลักที่จะช่วยให้เราออกแบบการทำงานของ Agent ได้อย่างเป็นระบบเหมือนกับการวาด Flowchart มาดูกันในหัวข้อลองไมค์วันนี้กันนะครับ  
  
โดยในหัวข้อนี้เราจะมาพูดถึงประเด็นเหล่านี้กัน  
\- ทำความรู้จักกับ LangGraph เปรียบเทียบความต่างกับ LangChain และทำความเข้าใจองค์ประกอบหลัก (State, Node, Edge)  
\- แนะนำการใช้งาน LangSmith Studio ตัวช่วย Visualize รูปแบบกราฟและ Debug Agent บน Local  
\- การเชื่อมต่อ Node ใน LangGraph เข้ากับ LLM และการบันทึกประวัติการสนทนาใน Thread ด้วย Persistence (Checkpoint)  
\- การทำ Human-in-the-loop เพื่อหยุดการทำงานของ Agent และรอการตัดสินใจ (Approve/Reject) จากมนุษย์ก่อนทำงานต่อ  
\- เพิ่มเติมเรื่อง Long Term Memory (Store) สำหรับจำข้อมูลข้าม Thread และ Time Travel เพื่อย้อนเวลาไปแก้ไข State ในอดีต

**6:21** · ไม่พูดว่าแรงเชนทำไม่ได้นะแต่ด้วยพื้นฐาน นองแรงกราฟอ่ะครับมันดีไซน์ออกมาเพื่อให้ ทำเรื่องนี้ได้โดยเฉพาะนะครับคือเราทุกคน อยู่บนพื้นฐานของโค้ดเนอไม่ว่าจะเป็น Python หรือ Ty Script คือเราฝืนทำอะไร บางอย่างคือมันทำได้อยู่แล้วเพียงแต่ว่า อะไรมันทำได้ง่ายมากกว่ากันแค่นั้นเอง เพราะฉะนั้นผมก็เลยมีประโยคหนึ่งเขียนไว้ ว่าถ้าแรงเชนน่ะมาตอบโจทย์อยู่แล้วคุณก็ ไม่มีเหตุผลอะไรที่ต้องมาแตะแรงกราฟเช่น เดียวกันนะครับแล้วก็อีกอันนึงที่ผมอยาก เล่าให้ทุกคนฟังก็คือแรงกราฟอ่ะมันมีความ

### องค์ประกอบของ LangGraph

**7:12** · องค์ประกอบของแรงกราฟกันบ้างครับเนี่ยถ้า ทุกคนพูดถึงความเป็นแรงกราฟเนอะมันจะแตก ต่างกับความเป็นแรงเชน 1 อย่างเวลาที่เรา คิดถึงงานแรงเชนเราจะคิดถึงแบบคอนเซปตลง ตรงไปตรงมาครับ agent ต้องต่อข้อมูลยังไง สร้างเครื่องมือยังไงใน agent ทำอะไร เพิ่มเติมได้บ้างมันจะเหมือนกับเวลาที่ คุณคิดงานของ Open Claw ที่มันมีความ สามารถในการที่จะไปไต่นู่นหรือพวกที่เป็น MCP พวกนี้แต่พอเป็นแรงกราฟอ่ะครับสิ่ง ที่เราจะคิดคือเราจะคิดเป็นงานเป็นสage เป็นโeครับเนี่ยเราก็จะคิดไปเหมือนกับ โฟชารเลยว่าอันนี้มีการเรียกฟังก์ชันยัง

**7:38** · ไงถ้าเกิดสมมุติว่าใช่ไปไงถ้าเกิดสมมุติ ไม่ใช่ไปยังไงเนี่ยมันก็จะเป็นโฟชารในภาพ นี้ผมจะขอแนะนำองค์ประกอบสำคัญของแรงกราฟ ก่อนและก็จะเป็นแกนสำคัญที่เราจะเล่ากัน ในหัวข้อนี้ด้วยครับองค์ประกอบสำคัญของ แรงกราฟครับคุณจะต้องนึกถึง 4 คำนี้ให้ ขึ้นใจก่อนนะครับเนี่ยซึ่ง 4 คำเนี้ยที่ ผมกำลังพูดถึงอ่ะครับมันคือการพยายามขยาย ภาพที่ทุกคนกำลังเห็นอยู่ตรงหน้านี้นะ ครับอย่างแรกสุดเมื่อกี้เราพูดกันแล้ว เนาะมันมี 3 คำที่คุณจะต้องนึกถึงแบบเป็น สัมพันธสำนึกเลยนะเวลาที่คุณใช้แรงกราฟ State Node กับอ่ะเรามาดูตามภาพเก่อน

**8:08** · Note ครับนิยามของมันคือฟังก์ชัน Python ที่ทำการบรรจุ logic ในการทำงานเอาไว้ ครับเพราะฉะนั้นถ้าเราพูดกันให้เข้าใจ ง่ายครับ Node คือ 1 ฟังก์ชันของการทำงาน ของ 1 Process ที่อยู่ภายในตัวของ workflow ครับอย่าง note 1 เนี่ยก็จะมี หน้าที่นึง 2 ก็จะมีหน้าที่นึง noe 3 ก็ จะมีหน้าที่นึงเป็นลักษณะประมาณนี้ครับที นี้ครับ no แต่ละอันสามารถที่จะเชื่อม เข้าหากันได้ยังไงนี่คือคุณสมบัติของสิ่ง ที่เรียกว่าครับจะเป็นตัวที่ระบุเส้นทาง ว่าหลังจากที่โหดนี้เสร็จไปยังโอยเช่นโหด

**8:37** · 1 เสร็จไปโหด 2 ต่อโเสร็จไปโหด 3 ต่อไอ้ นี่ไปโหด n ต่ออันมันจะมีสตาร์ทกับ n ด้วยซึ่งในแรงกราฟครับมันจะมีโหนดพิเศษก็ คือโหดสตาร์ทกับ n เพื่อเป็นตัวที่บอกว่า เริ่มต้นจากตรงไหนสิ้นสุดจากตรงไหนออกมา อย่างเช่นเคสเนี้ยเรามีการประกาศเนอว่า อ่ะเราให้ no start ทำการเชื่อมไปยังโหด 1 ก็คือสตาร์ทเริ่มต้นปุ๊บไปยังโหลด 1 เป็นโหดแรกไอ้การ define สิ่งที่เรียกว่า age อ่ะครับก็คือเป็นการระบุเส้นทางว่า จากสตาร์ทไปยังโเนี่ยก็คือเส้นทางที่มัน สามารถไปได้ทีนี้อย่าง age มันก็จะมี

**9:04** · คอนเซปอยู่ด้วยว่าเอ้ยถ้าเกิดมันมีมาก กว่า 1 เส้นทางเป็นแบบเนี้ยโe 2 3 มันก็ จะมีเรื่องของคitionในการที่จะมากำหนดอีก ทีนึงว่ามันจะเลือกไปโหดที่ 2 กับโหนดที่ 3 ด้วยเหตุผลอะไรก็จะเป็นเหมือนกับภาพบน เนี้ยที่มันก็จะมีสิ่งที่เป็นเหมือนกับ คitionอีกทีนึงที่หลังจากที่มันเช็คเสร็จ ปุ๊บมันก็จะเลือกว่าอาจจะไปโหนเครื่องมือ

**9:20** · หรือจะไปโหนรีรทนะครับซึ่งเดี๋เรามีตัว อย่างให้ทุกคนดูในวันนี้ด้วยนะครับนี่คือ สิ่งที่ 2 ครับคือครับเพราะฉะนั้น Node คือฟังก์ชันการทำงานเปรียบได้เป็น 1 กับ เนี่ย 1 ก้อนของการทำงานที่อยู่ภายใน workflow age ครับคือเส้นทางการเชื่อม ต่อว่าโหดไหนสามารถเชื่อมมายังโหดไหนได้ บ้างและสุดท้ายครับเพื่อให้ข้อมูลทั้ง workflow ครับสามารถที่จะแชร์ข้อมูลร่วม กันได้ครับจะต้องมีสิ่งหนึ่งที่เรียกว่า state ครับ state ครับจะเป็นหนึ่งในส่วน ที่สามารถส่งข้อมูลไปมาหากันได้ภายใน

**9:47** · ระหว่างโตอย่างเช่นสมมุติว่าคุณจะขนของ บางอย่างออกไปอ่ะแล้วคุณเดินอ่าคุณมีการ procสตั้งแต่โหด 1 ก่อนสมมุติว่าคุณต้อง ขนข้อมูลบางอย่างไปก่อนแล้วข้อมูลเนี้ย ต้องเอาไปใช้ในโหดที่ 3 สมมุตินะครับ สมมุติสิ่งที่คุณจะต้องทำก็คือคุณจะต้อง เอาข้อมูลจากโหด 1 อ่ะใส่สageไปแล้วทอไป ถึงโหดที่ 3 ครับมันก็จะดึงข้อมูลจากสage ที่คุณมีการเก็บเอาไว้อ่ะนำสageนั้นนะ ครับมาใช้งานแล้วก็สามารถที่จะเก็บคำตอบ เสร็จสageเพื่อไปส่งเป็นอันดับสุดท้าย แล้วก็ทำการคืนเป็นสageผลลัพธ์ออกไปได้นะ

**10:12** · ครับซึ่งสageน่ะมันจะเป็นการกำหนดเป็น เชิงของข้อมูลว่าหน้าตาของข้อมูลจะเป็น ประมาณไหนในการที่จะรับแล้วก็ในการที่จะ ส่งออกเข้าไปนะครับไม่ใช่ส่งออกเข้าเดี๋ ต้องส่งออกไปเพราะฉะนั้นครับสageอ่ะมัน คือส่วนกลางที่ใช้สำหรับการแสดงผลข้อมูล ปัจจุบันแล้วก็การเก็บข้อมูลของสage ปัจจุบันที่ผ่านไปยังโตต่างๆซึ่งก็จะเป็น ทั้งตัวที่สามารถใช้สับส่งต่อข้อมูลรวม ถึงตัวที่สามารถแสดงผลข้อมูลที่เป็นแบบ ข้อมูลณปัจจุบันที่มีการส่งต่อเข้ามาด้วย

**10:37** · นะครับซึ่งความเป็นสageเนี่ยเราก็จะทำการ กำหนดด้วยสิ่งที่เรียกว่า class ของตัว ที่เป็นสageออกมาซึ่งมันก็จะมีแพทเทิร์น ในการประกาศอยู่แล้วนะครับว่ามันควรจะ ต้องประกาศเป็นลักษณะอันไหนออกมานะครับ องค์ประกอบสุดท้ายครับ persistance กับ checkpoint ครับเพื่อให้ฝั่งของ workflow ครับเนี่ยภาพที่เรากำลังเห็นทั้งหมดเนี่ย ทั้งก้อนเนี้ยครับที่เรากำลังพูดถึงเมื่อ กี้ State Note พวกเนี้ยเนี่ยคือ 1 workflow ของการทำงานครับเพื่อให้ workflow อ่ะครับมันมีความสามารถในการ ที่จะจดจำเอาไว้ได้ว่าเอ้ยมันเคยคุยข้อ มูลในเทรดนี้ไว้ยังไงมันเคยคุยไปจนถึงจุด

### Example 2.2 - เพิ่มความสามารถ Human in the loop เข้าไป

**44:58** · บางอย่างแล้วกลับมาทำต่อเนื่องได้อย่าง เช่นคุณมีแคชับางอย่างคุณต้องการหยุดก่อน เพื่อรอดูว่า user จะตอบมาว่าไงแล้วคุณจะ ค่อยไปต่อเราสามารถที่จะใช้คุณสมบัติของ เช็คพ์ครับในการที่จะดำเนินการต่อไปได้ โดยที่ตัวแรงกราฟจะเป็นคนจัดการให้เราทุก คนคือคุณไม่ต้องไปเขียนแบบเอ้ยค่อยมา continue ต่อเลยอะไรประมาณนี้แรงกราฟมี คอเซปที่ทำให้คุณสามารถอลับกับรีซูomได้ ผ่านตัวอย่างที่ผมกำลังจะแชร์ต่อไปนี้ ครับอย่างแรกสุดครับผม import ของทั้งหมด เข้ามาครับทีนี้ผมจะทำการเล่นของบางอย่าง

**45:25** · ให้ทุกคนดูครับผมทำการ define 2 stage ขึ้นมาครับปกติก่อนหน้านี้เรา define stage เดียวนะคราวนี้เรา define 2 stage และ agent ที่เราจะทำในหัวข้อนี้ ครับเราจะทำ agent ที่ชื่อว่า approve action ครับ concept ของ agent เนี้ยมี อยู่ว่าเราจะทำการรับแtionบางอย่างมาครับ แลหลังจากนั้นครับเราจะรับคำสั่งมาว่าจะ approve การกระทำนี้หรือ โดยถ้าเกิด approve ยืนยันโดยการส่ง true

**45:47** · ถ้าเกิด reject ยืนยันโดยการส่ง false หรืออะไรก็ได้ที่ไม่ใช่ true ออกมาเพราะ ฉะนั้นครับสิ่งที่เราทำครับจังหวะ input อ่ะเราอยากให้มันใส่เป็นแค่อย่างเดียวคือ เราอยากให้มันใส่แค่ action อย่างเดียว แต่ว่าตอนที่เรามีการทำจริงอ่ะเราจะมีการ กำหนดสิ่งที่เรียกว่าสเตตัสเริ่มต้นของ ตัวที่เป็นสageด้วยเพราะฉะนั้นครับในเคส เนี้ยมันเลยจะเกิดเป็น 2 อย่างขึ้นมา 1 ก็คือสageที่เรารับ 2 ก็คือสageที่เราจด

**46:08** · จำหรือใช้ภายใน input state ครับคือข้อ มูลที่เรารับมาเรารับแค่actionัอย่าง เดียวออกมาแต่ทุกแtionที่เรามีการสร้างมา ใหม่ครับเราจะสร้างมาใหม่เป็น approve state เพื่อให้มันมีสเตตัสเริ่มต้นเป็น pending ขึ้นมาได้นะครับซึ่งเคสเนี้ย ครับเพื่อให้มันสามารถที่จะเรียกใช้งาน ภายในได้ง่ายผมเลยจัดการเปลี่ยนมาเป็น Data class เดี๋ให้ทุกคนได้เห็น field ของ Data class นะครับจริงๆ Data Class กับ Typ อ่ะมีความใกล้เคียงกันมากวิธีการ

**46:29** · ใช้ก็แตกต่างกันนิดหน่อยคำสั่ง Manage ก็ จะแตกต่างกันนิดหน่อยแต่ผมสร้างเป็น Data class เพื่อให้ทุกคนน่ะได้เห็นภาพด้วย ว่าในกรณีที่ใช้คลาสภายในคุณอาจจะใช้ เพียงแค่ Data Class เพื่อให้มันสามารถ assess เข้าถึงกันได้ง่ายเมื่อเทียบกับ เวลาที่เป็น dictionary ในหลายๆเคสนะครับ หลังจากนั้นครับไอ้ action detail เหมือนกันแต่เพิ่มเติมก็คือเพิ่มสตัสออก มาเป็น optional ซึ่งก็จะมีการเรียงเป็น pending approve reject แล้วก็ default ให้เป็น pending ไปจากนั้นครับอ่ะผมกำหนด

**46:51** · ตงนี้ก่อนฟังก์ชันเราทำการสร้างโหด ฟังก์ชันมาทั้งหมด 3 อันครับโeดฟังก์ชัน อันแรกครับคือ approval node ครับคือโหด ที่มันจะเข้ามาเสมอในทุกครั้งที่มันมีการ ส่งแtionมาครับทุกคนสังเกตเห็นอะไรนะครับ ตอนที่ผมมีการรับ input เข้ามาในรอบนี้ ครับผมเปลี่ยนมาเป็น approve state แทน ครับหมายความว่าสิ่งที่ตัวกราฟจะต้องทำ อ่ะครับก็คือกราฟอ่ะจะต้องทำการ convert

**47:09** · จาก input state นะครับมาเป็น approve stage ตัวนี้ออกมาซึ่งความสามารถของแรง กราฟครับสามารถที่จะ map key แล้วก็ สามารถที่จะมาใช้งานร่วมกันได้นะครับผ่าน คำสั่ง builder ซึ่งเดี๋เรา define อีกที นึงให้ทุกคนเห็นนะครับแต่หลักๆก็คือทุกคน ที่เข้ามาครับเริ่มต้นเราจะเข้ามาที่ approve nde ตัวนี้ออกมาก่อนผมจำเป็น ต้องวาดภาพเนี่ยผมว่าวาดภาพหน่อยก็ดีเออ เดี๋ยวมันจะลอยๆเกินทุกคนเดี๋มันจะลอยๆ เกินสตาร์ทอ่ะอย่างแรกสุดครับข้อมูลทั้ง หมดอ่ะเราจะต้องเข้ามาอันนี้ผมวาดผมวาด

**47:33** · สั้นๆอย่างงี้นะ Action ส่งเข้ามาผ่าน Approval Note แล้วหลังจากนั้นครับไอ้ Approval Node เนี่ยหลังจากที่มันเข้า มาครับสิ่งที่มันจะเกิดขึ้นก็คือมันจะมี คำสั่งหนึ่งครับชื่อว่า Interrupt ครับ เมื่อไหร่ก็ตามครับที่แรงกราฟครับเจอคำ สั่งอรupเมื่อไหร่ครับสิ่งที่มันจะเกิด ขึ้นคือมันจะทำการเบรค process ไว้ตรงนี้ และมันจะค่อยรีซูomต่อก็ต่อเมื่อได้รับ สิ่งหนึ่งที่เป็นคำสั่งที่ชื่อว่า Command เข้ามาแทนครับหมายความว่าไงครับหมายความ ว่าสิ่งที่เกิดขึ้นก็คือตอนที่มาถึง Approval Node นะครับสิ่งเกิดก็คืออี ตลับตรงเนี้ยจะมีการส่งข้อความบางอย่าง กลับไปยังฝั่งที่เป็นแบบของ S agent ที่

**48:04** · เรามีการรับตัวแปรเอาไว้แล้วหลังจากนั้น ครับสิ่งที่เราต้องทำก็คือเราทำการ Invoke ซ้ำพร้อมกับการส่ง Command เข้ามาว่าคุณ จะให้ Action เนี้ยไปต่อหรือไม่นะครับและ action เนี้ยก็จะทำการรับมาผ่านตัวแปร ที่ชื่อว่า decision ที่ถ้า decision เป็น to command go to proceed แต่ ถ้าเกิด else command go to cancel proceed cancel ตัวนี้คืออะไรนะครับตัว นี้คือชื่อโeที่มันจะไปต่อครับหมายความ ว่าไอ้ความสามารถของตัวเนี้ยถ้าเราลองดู นะอ่าอันนี้คือชื่อฟังก์ชันส่วนอันนี้ก็ คือ return ฟังก์ชันของตัว Python นะครับ เราอ่ะรับที่ชื่อ approval node มารับ

**48:36** · ตัวที่เป็น approval state มาแล้ว return ที่เราจะทำคือเราต้องทำ command 1 ใน 2 อย่างนี้คือ proc กับ cancel เพราะฉะนั้น ครับสิ่งที่เราจะต้องทำก็คือเราจะต้องตอบ ตัวที่เป็น interlup ตัว decision ตัวนี้ ครับแล้ว return ออกมาเป็น true หรือ false ออกมาเพื่อให้มันสามารถ command ไปสู่ proceed กับ cancel ออกมาได้เพราะ ฉะนั้นนะครับถ้าเราลองวาดเป็นตัวเนี้ยมัน ก็จะเหมือนกับเราวางคitionบางอย่างออกไป ในโหดตัวเครับแล้วเราบอกว่า decision decision เไปต่อมยนี่หลังจากอิตลับนะ หลังจากอิตดับซึ่งสมมุติว่าเป็น true เป็น true ไปนี่เป็น 4 มา cancel แล้ว หลังจากนั้นครับทั้ง 2 ตัวก็จะมาจบที่ n

**49:08** · เป็นอันดับสุดท้ายเป็นภาพประมาณนี้ครับ ทุกคนเพราะฉะนั้นงั้นถ้าเราลองดูตามภาพ เนี้ยสิ่งที่เราต้องเชื่อมคืออะไรครับอัน นี้มันเป็น conditionition ใน node เนาะ สิ่งที่เราต้องเชื่อมก็คือเราต้องเชื่อม start approval node และ appal node ต้องสามารถไปยัง 2 ที่ก็คือ process cancel ออกมาได้และ process cancel ต้องสามารถไปสู่จบออกมาได้อ่ะเพราะฉะนั้น

**49:25** · เราต้องค่อยๆทำไปทีละลำดับนะทุกคนนี่ซึ่ง process กับมันต้อง proceed ดิเออ proceed ขออภัยทุกคน proceed proceed proceed ผมชอบยอมรับนะผมจะชอบสับสนกับคำ นี้เหลือเกินอ่ะ proceed ก็คือดำเนินการ นะครับคือดำเนินการก็ทำการ return status เป็น approve แต่ถ้าเกิด cancel node ก็ return สตัสเป็น reject ซึ่งก็เป็น คุณสมบัติสเตตัสของตัวที่เป็น approval state ตัวนี้ออกมานะครับอ่ะเริ่มต้นครับ builder ครับเราประกาศตัวที่เป็น state กราฟขึ้นมาครับ approval state input เท่ากับ input stage แบบนี้เอ้าเล่นออก มาให้หมดเลยเหรออ่ะตามนี้ตามนี้เนี่ยเห็น

**49:53** · มั้ทุกคนมันก็เดาๆได้แหละว่าเราจะสร้าง โหดประมาณไหนจากคitionนะครับอ่ะสิ่งที่ เราทำการประกาศครับคราวเนี้ยเราประกาศ 2 อย่างเพิ่มเข้ามาครับ 1 ก็คือเราประกาศ ตัวที่เป็นสageกราฟรับ stage เข้าไป เหมือนเดิมแต่เพิ่มเติมคือเรา speit ตัว ที่เป็น input state เข้าไปครับว่าเป็น input stage มาเป็นตามนี้แล้วแล้วก็ overall state รวมถึง output state ที่เราใช้เราก็จะใช้เป็น approval state ตัวนี้เข้ามาหมายความว่าตอน input น่ะเรา รับแค่ key เดียวเข้ามาก็จริงแล้วหลังจาก นั้นครับ map key เข้ามาเป็น approval state แล้วหลังจากนั้นก็ใช้ approval stage จนจบการทำงานทั้งหมดออกมานั่นเอง

**50:23** · นะครับอันเนี้ยก็คือวิธีการประกาศ state กราฟในกรณีที่คุณต้องการให้ input กับ overall อ่ะแตกต่างกันและมันก็จะมีด้วย นะกรณีที่อยากให้ input output overall แตกต่างกันก็จะมีวิธีการประกาศแบบนี้รวม ถึงถ้าเกิดกรณีที่มีการแชร์สageภายในเกิด ขึ้นมันก็จะมีวิธีการประกาศอีกแบบนึง เหมือนไว้ค่อยว่ากันในเคสหลังๆนะครับเดี๋ เราอาจจะมี agent ที่มันมีความลึกซึ้ง ขึ้นในซีรียส์นี้นะครับแต่ตอนนี้ผมให้ทุก คนรู้จักประมาณนี้ก่อนเพื่อจะได้ไม่งงจน เกินไปนะทุกคนอ่ะ 3 โหดเมื่อกี้ที่เราพูด
